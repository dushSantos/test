# Инструкция по сохранению товаров в репозиторий

## Проблема
Товары, добавленные через админ-панель, сохраняются только в localStorage браузера, поэтому они видны только на том компьютере, где были добавлены.

## Решение

### Вариант 1: Автоматическое сохранение через GitHub API (Рекомендуется)

1. **Создайте GitHub Personal Access Token:**
   - Перейдите: https://github.com/settings/tokens
   - Нажмите "Generate new token (classic)"
   - Выберите права: `repo` (полный доступ к репозиторию)
   - Скопируйте токен

2. **Настройте переменные окружения в Vercel:**
   - Откройте проект на Vercel
   - Перейдите в Settings → Environment Variables
   - Добавьте:
     - `GITHUB_TOKEN` = ваш токен
     - `GITHUB_REPO` = `ваш-username/название-репозитория` (например: `username/ZipSeoul`)
     - `GITHUB_BRANCH` = `main` (или ваша ветка)

3. **После настройки:**
   - При добавлении товара через админ-панель, он автоматически сохранится в репозиторий
   - После деплоя на Vercel товары будут видны всем пользователям

### Вариант 2: Ручное сохранение через экспорт файла

1. **Добавьте товары через админ-панель**

2. **Нажмите кнопку "Экспорт товаров"** в админ-панели

3. **Скачанный файл** (`zipseoul-products-XXXXX.json`) загрузите в корень проекта как `products-data.json`

4. **Закоммитьте и запушьте изменения:**
   ```bash
   git add products-data.json
   git commit -m "Update products"
   git push
   ```

5. **После деплоя на Vercel** товары будут видны всем пользователям

## Как это работает

- Сайт сначала пытается загрузить товары из файла `products-data.json`
- Если файл не найден, используется localStorage (для локальной разработки)
- Все страницы (`category.html`, `product.html`, галерея) автоматически загружают данные из файла

## Важно

- Файл `products-data.json` должен находиться в корне проекта
- После каждого обновления товаров нужно либо:
  - Использовать автоматическое сохранение (Вариант 1)
  - Или экспортировать и загрузить файл вручную (Вариант 2)
