# Настройка админ-панели для сохранения товаров в репозиторий

Для того, чтобы товары, добавленные через админ-панель, сохранялись в код проекта и были видны всем пользователям, необходимо настроить интеграцию с GitHub API.

## Вариант 1: Использование Vercel Serverless Functions (Рекомендуется)

### Шаг 1: Создайте GitHub Personal Access Token

1. Перейдите в GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Нажмите "Generate new token (classic)"
3. Выберите права доступа:
   - `repo` (полный доступ к репозиторию)
4. Скопируйте созданный токен

### Шаг 2: Настройте переменные окружения в Vercel

1. Перейдите в настройки вашего проекта на Vercel
2. Откройте раздел "Environment Variables"
3. Добавьте следующие переменные:
   - `GITHUB_TOKEN` - ваш Personal Access Token
   - `GITHUB_REPO` - формат: `username/repository-name` (например: `yourusername/ZipSeoul`)
   - `GITHUB_BRANCH` - ветка (по умолчанию: `main`)

### Шаг 3: Создайте папку api

Убедитесь, что файл `api/save-products.js` находится в корне проекта.

### Шаг 4: Деплой

После настройки переменных окружения, перезапустите деплой на Vercel.

## Вариант 2: Использование Firebase/Supabase (Альтернатива)

Если вы не хотите использовать GitHub API, можно использовать Firebase или Supabase для хранения данных.

### Настройка Firebase:

1. Создайте проект в Firebase
2. Включите Firestore Database
3. Обновите `admin.js` для работы с Firebase вместо localStorage

## Вариант 3: Простой бэкенд на Node.js

Создайте простой сервер на Node.js, который будет принимать данные и сохранять их в файлы.

## Текущая реализация

Сейчас товары сохраняются:
- В localStorage (локально в браузере)
- Через API endpoint `/api/save-products` (если настроен) - сохраняет в GitHub репозиторий

После настройки GitHub API, все товары будут автоматически коммититься в репозиторий и доступны всем пользователям после деплоя.
